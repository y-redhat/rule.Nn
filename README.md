  <h1>チームルール</h1>

  <h2>第1条: 部門</h2>
  <ul>
      <li>1.1 部門は以下の4つとする。</li>
      <li>Red Team部門</li>
      <li>サーバー部門</li>
      <li>Js(Ts)部門</li>
      <li>HTML/CSS部門</li>
  </ul>

  <h2>第2条: 管理職</h2>
  <ul>
      <li>2.1 チームには以下の管理職を設ける。</li>
      <li>管理人: チーム全体の運営を統括する。</li>
      <li>副管理人: 管理人をサポートし、管理人不在時にその役割を代行する。</li>
  </ul>

  <h2>第3条: 権限</h2>
  <ul>
      <li>3.1 部門新築の権限は管理人および副管理人に限る。</li>
      <li>3.2 ノートの作成権限はすべてのメンバーに与えられる。</li>
      <li>3.3 アナウンス権限はすべてのメンバーに与えられる。</li>
      <li>3.4 大事なノートの作成権限はすべてのメンバーに与えられる。</li>
      <li>3.5 メンバーの強制退会権限は管理人および副管理人に与えられる。ただし、2/3の可決が必要であり、副管理人と管理人はこのプロセスの対象外とする。</li>
      <li>3.6 ノートの削除権限は管理人および副管理人に与えられる。</li>
      <li>3.7 メンバーの追加権限はすべてのメンバーに与えられる。</li>
  </ul>

  <h2>第4条: 特別ルール</h2>
  <ul>
      <li>4.1 メンバーの退会は2/3の可決が必要であり、副管理人と管理人はこのプロセスの対象外とする。ただし、管理層（管理人および副管理人）は投票には参加できるが、このシステムの議決によって退会されることはない。</li>
      <li>4.2 副管理人と管理人は、互いに管理層の1/2の可決があればその地位を失う。管理人は副管理へ、副管理人は一般メンバーへ降格する。この会議にはすべてのメンバーが申し込むことができ、申し込む際はノートに書くか、イベントを作ることで申し込むことができる。</li>
      <li>4.3 最初の管理層は、最初のプロジェクト（チェスゲーム）の構築に携わったメンバーが初代管理層となる。後に管理層を増やすことは可能であり、減らすことも可能だ。</li>
      <li>4.4 副管理人を増やす場合は以下の条件を満たす必要がある。</li>
      <li>複数のプロジェクトを同時進行している場合。</li>
      <li>ものすごい貢献をしたと管理層（管理人、副管理人）が決定した場合。</li>
  </ul>

  <h2>第5条: メンバーとしての認定</h2>
  <ul>
      <li>5.1 メンバーとして認められるためには、LINEのオープンチャット「Nn（仮）」に参加する必要がある。</li>
  </ul>

  <h2>第6条: 喧嘩や不祥事の発覚</h2>
  <ul>
      <li>6.1 喧嘩や不祥事が発覚した場合、喧嘩をした両者には投票権が与えられる。</li>
      <li>6.2 全体投票により、以下の決定が行われる。</li>
      <li>2/3の可決で退会</li>
      <li>1/2の可決で活動停止（期間: 半年）</li>
      <li>6.3 この際、喧嘩した両者は審判の対象となり、その結果に基づいて処置が決定される。</li>
  </ul>

  <h2>第7条: 日本国憲法の遵守</h2>
  <ul>
      <li>7.1 日本国憲法を守ることをチームの基本方針とする。</li>
      <li>7.2 やむを得ない場合は個人の判断に委ねる。ただし、このチームが犯罪行為に直接関与していない限り、その責任は当事者以外は取らない。</li>
  </ul>

  <h2>第8条: ハッキングおよびサイバー攻撃に関するルール</h2>
  <ul>
      <li>8.1 ハッキングやサイバー攻撃に関しては、個人の行動として行われた場合、その責任は取らない。</li>
      <li>8.2 チームとしての活動において、ハッキングやサイバー攻撃をするかについての判断は全体で決める。やむを得ない場合のみ実行する</li>
    　<li>8.3　一人でも否定意見もしくは中立意見の場合、チームでの実行とはしない。代わりに実行する場合は、チーム全員に責任がある。</li>
  </ul>

  <h2>第9条: ルール違反の対応</h2>
  <ul>
      <li>9.1 ルールを破っている人を発見した場合は、ノートに「警告⚠: 内容」と記載し、調査が入る。</li>
      <li>9.2 調査の結果、ルール違反が確認された場合、適切な処置が取られる。</li>
  </ul>

  <h2>第10条: 方針決定</h2>
  <ul>
      <li>10.1 チームの方針は2/3の可決によって決定される。</li>
  </ul>

  <h2>第11条: その他の運営ルール</h2>
  <ul>
      <li>11.1 定期的なミーティングを設定し、チームの進捗状況を確認する。</li>
      <li>11.2 SlackやDiscord、LINEオープンチャットなどのコミュニケーションツールを使用して、常に連絡を取り合うことを推奨する。</li>
      <li>11.3 各部門は定期的にプロジェクトの進捗を報告し、フィードバックを受ける。</li>
      <li>11.4 参加しないメンバーにはリマインダーを送り、参加を促す。</li>
      <li>11.5 メンバーは自分の部門にあったサブトークルームに入ることができる。</li>
      <li>11.6 人手不足の場合、管理層の権限はすべての部門に適応されるが、人員が確保でき次第、副管理人は部門ごとの部門管理人となり、管理人は最高指導権を持つ。</li>
  </ul>

  <h2>第12条: ルール変更について</h2>
  <ul>
      <li>12.1 このルールは最初に管理者によって制定される。</li>
      <li>12.2 異議申し立てがある場合は管理者に伝え、再検討される可能性がある。</li>
      <li>12.3 このルールはいつでも変更する可能性がある。</li>
      <li>12.4 ルールのドメインはいつでも変更する可能性がある。</li>
      <li>12.5 変更した際には必ず一週間の猶予があり、その間には伝えることとする。</li>
  </ul>
